# Release 0.3.0: Inputs

Status: Complete

## Goal

Make the Source Material stage functional and durable: script text, narration
audio, visual references, and pacing persist with the selected video.

## Included

- [x] Input contracts and migration scope
- [x] SQLite migration for video inputs and imported assets
- [x] Script autosave and text-file import
- [x] Narration audio import, replacement, and removal
- [x] Multiple visual-reference imports and removal
- [x] Controlled copies inside the video project directory
- [x] Scene pacing persistence
- [x] Input validation and readiness summary
- [x] Rust and frontend tests

## Explicitly excluded

- Audio transcription
- Word-level timestamps
- Visual-plan generation
- Reference-image analysis
- Provider calls

## Acceptance criteria

- Input state survives native application restarts.
- Imported source files are copied; external originals are never modified.
- Unsupported file types and oversized text files are rejected.
- Visual-plan continuation requires both a non-empty script and narration audio.
- All input mutations update video modification time and recovery snapshots.

- Four frontend tests passed

- Optimized executable launch smoke test passed
