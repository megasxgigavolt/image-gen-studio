# Release 0.2.0: Projects

Status: Complete

## Goal

Replace the mock home workspace with durable local channels and videos that
survive application restarts and can be recovered after mistakes.

## Included

- [x] Release scope and persistence contracts
- [x] SQLite database created under the application data directory
- [x] Versioned schema migrations
- [x] Channel creation and listing
- [x] Video creation, listing, opening, and resume state
- [x] Application-controlled project asset directories
- [x] Recoverable channel and video trash
- [x] Automatic video snapshots with ten-snapshot retention
- [x] React home screen backed by native commands
- [x] Creation dialogs and empty/loading/error states
- [x] Rust repository tests and frontend tests

## Explicitly excluded

- Script/audio/reference persistence
- Visual-plan persistence
- Provider credentials
- AI calls
- Cloud sync
- Project bundle export

## Acceptance criteria

- Data persists across native application restarts.
- Creating a channel creates its controlled project directory.
- Opening a video updates the last-resume location.
- Deleted channels and videos disappear from normal lists but remain recoverable.
- Snapshot retention never exceeds ten automatic snapshots per video.
- Database migrations and repository behavior are covered by Rust tests.
