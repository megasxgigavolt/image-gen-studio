# Release 0.8.0: Export and Portability

Status: Complete

## Goal

Make finished stills and complete local projects portable, inspectable, and
recoverable without exposing provider secrets.

## Included

- [x] Export latest image versions to a user-selected folder
- [x] Export all latest stills with deterministic names and manifest
- [x] Create a portable project bundle with project metadata and assets
- [x] Import and validate a project bundle without overwriting existing data
- [x] Bundle integrity checks and clear recovery errors
- [x] Windows installer verification
- [x] Tests and optimized native verification

## Acceptance notes

- `.agsproj` bundles are ZIP archives with a versioned manifest and controlled assets.
- Imports validate every declared asset before creating a collision-safe imported project.
- Unsafe archive paths are rejected and failed imports clean up created rows and folders.
- API credentials and global application settings are never exported.

## Explicitly excluded

- Cloud sync or accounts
- Provider credentials in exports
- Video timeline rendering
- Publishing and channel integrations

## Invariants

- Exports never include API keys.
- Imports receive fresh local identifiers when collisions exist.
- Existing projects are never overwritten silently.
- Partial import failures leave no half-created project.
