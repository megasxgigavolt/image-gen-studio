# Release Roadmap

Each release must remain usable, testable, and narrowly scoped. Work not listed
in a release is not silently added to it.

| Release | Theme | Outcome |
| --- | --- | --- |
| 0.1.0 | Foundation | Complete: runnable native shell, architecture, design system, navigation, CI-ready quality gates |
| 0.2.0 | Projects | Complete: channel/video creation, SQLite persistence, resume, trash, snapshots |
| 0.3.0 | Inputs | Script/audio/reference intake and validation |
| 0.4.0 | Visual Plan | Transcription adapter, sentence timing, AI plan, drag/drop, reset |
| 0.5.0 | Image Workspace | Three-layer prompts, settings, single generation, history |
| 0.6.0 | Bulk Jobs | Concurrent generation, retries, pause/stop/resume, recovery |
| 0.7.0 | Image Editing | Instruction-based editing and version comparison |
| 0.8.0 | Export and Portability | Project bundles, still export, recovery hardening |
| 0.9.0 | Timeline Foundation | Interactive editor data model and disabled feature-preview surface |
| 1.0.0 | Personal Stable | Hardened Windows portable release |

Animation generation remains a placeholder before `1.0.0`. Full video editing,
captions, thumbnails, metadata, publishing, cloud sync, accounts, teams, and
commercial licensing follow the stable personal release.
