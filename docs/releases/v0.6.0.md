# Release 0.6.0: Bulk Jobs

Status: Complete

## Goal

Turn per-still generation into durable, recoverable bulk jobs without changing
the image-editing boundary established for v0.7.0.

## Included

- [x] Persistent bulk job and job-item SQLite migration
- [x] Generate pending stills from the image workspace
- [x] Gemini concurrency limit of two active requests
- [x] Per-item progress, attempts, and provider error history
- [x] Exponential retry for provider failures
- [x] Pause, resume, and stop controls
- [x] Startup recovery for interrupted jobs
- [x] Tests, optimized native build, and release verification

## Acceptance notes

- Two independent workers process queued stills while the UI remains navigable.
- Jobs and item outcomes survive application restarts; interrupted work reopens paused.
- Pause and stop prevent new items from starting. Active provider calls finish safely.
- Completed renders remain available when another item exhausts its retries.

## Explicitly excluded

- Image editing or inpainting
- Reference-image conditioning
- Prompt chat or automatic prompt rewriting
- Export bundles and timeline editing

## Invariants

- Navigation never cancels a running job.
- A stopped job does not start new items.
- Completed renders remain durable if later items fail.
- Provider keys remain inside the Rust boundary.
- Each job item references an immutable prompt version.
