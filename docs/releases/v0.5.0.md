# Release 0.5.0: Image Workspace

Status: Complete

## Goal

Provide durable three-layer prompts, secure provider configuration, single-image
generation, and per-still version history.

## Included

- [x] Provider and legacy workflow audit
- [x] Windows Credential Manager integration
- [x] Configurable OpenAI and Gemini models
- [x] Three-layer image settings
- [x] Prompt and render version history
- [x] Single Gemini image generation
- [x] Functional Prompt and Settings tabs
- [x] Tests and native verification

## Acceptance notes

- Provider secrets never leave the Rust boundary and are stored through Windows Credential Manager.
- Each render creates an immutable prompt snapshot and is written below the video's controlled `renders/<still-id>` folder.
- Gemini generation uses the configured model and reports provider errors without creating false render history.
- Bulk generation, retries, editing, and reference-image conditioning remain excluded.

## Explicitly excluded

- Concurrent bulk generation
- Image editing/inpainting
- Chat refinement
- Reference-image conditioning
